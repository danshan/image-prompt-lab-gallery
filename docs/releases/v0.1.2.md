# Image Prompt Lab v0.1.2

## Highlights

- Adds Gallery version tree inspection and version promotion.
- Redesigns Gallery and Albums interaction boundaries so Gallery stays an all-assets browser while Albums owns collection workflows.
- Completes a broad DDD architecture refactor that moves business decisions out of runtime adapters.

## User-facing changes

- Added Gallery version tree support for inspecting asset lineage.
- Added promote-as-new-asset behavior for selected versions.
- Redesigned album filtering around explicit Gallery filters instead of implicit album selection state.
- Improved Album management and add-to-album flow ownership.
- Updated README status to describe the current desktop release baseline.

## Fixes

- Fixed several CI and Linux release build issues for the Tauri desktop release workflow.
- Fixed clippy warnings and workflow defaults needed for reliable release validation.
- Installed required Linux dependencies for Tauri CI jobs.

## Developer and release notes

- Split Gallery read models for search, filters, details, version trees, task origins, and cards.
- Routed Tauri tag and album operations through core application use cases.
- Removed daemon access to core task service internals.
- Added library lifecycle use case boundaries and clarified runtime refresh ownership.
- Archived the DDD refactor completion audit and synced OpenSpec specs.

## Install

Download the attached macOS asset from this release. Current builds use macOS ad-hoc signing plus Tauri updater signing, not Apple Developer ID notarization, so the first launch may require the standard macOS Gatekeeper override.

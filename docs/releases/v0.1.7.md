# Image Prompt Lab v0.1.7

## Highlights

- Hardens desktop daemon startup so missing sidecar binaries fail with an actionable diagnostic instead of a generic `os error 2`.
- Bounds background automation daemon service control so Settings `Restart` and `Repair` cannot block indefinitely on `launchctl`.

## User-facing changes

- Settings Automation now reports recoverable daemon service-control failures instead of leaving the workflow stuck.
- Generate Queue and Review regeneration daemon startup now fail fast when the local daemon binary is missing.

## Fixes

- Fixed daemon binary discovery to validate explicit and fallback paths before sidecar startup.
- Fixed packaged-app daemon discovery to include the macOS app bundle resources location.
- Fixed Automation daemon `launchctl` calls to return a timeout diagnostic when the OS service command does not complete promptly.

## Developer and release notes

- Archived the `fix-daemon-sidecar-startup` OpenSpec change.
- Synced the daemon startup failure requirement into the main `task-manager-daemon` spec.
- Updated the release build hook and Tauri external sidecar packaging so the daemon is built with the release profile before desktop packaging.

## Install

Download the attached macOS asset from this release. Current builds use macOS ad-hoc signing plus Tauri updater signing, not Apple Developer ID notarization, so the first launch may require the standard macOS Gatekeeper override.
